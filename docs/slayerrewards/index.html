MCSlayerRewards

A standalone plugin that pays players for slaying mobs — configurable money ranges, custom item drops, and holographic feedback at the kill location. Payouts settle through MCEconomy.

io.github.mcclauneck.slayerrewards SemVer 1.0.0 depends: MCIdentity, MCEconomy
Change log Repository ↗

Highlights

⚔️
Per-mob rewards. Give any mob a fixed or random money range and a currency, in a plain YAML file.
🎁
Custom drops. Add Base64 items with per-drop chances, and optionally cancel vanilla drops.
✨
Holographic feedback. A +amount currency TextDisplay pops at the death location on a payout.
🖱️
GUI editor. Manage loot tables and drop chances entirely in-game.
🗄️
Five storage backends. SQLite (default), MySQL, PostgreSQL, MongoDB, or the central REST server for kill/reward logs.

Module architecture

A multi-module Gradle build modelled on MCEconomy. The universal engine jar shades every module and the thin platform entry points into one plugin.

ModuleJava packageResponsibility
apiio.github.mcclauneck.slayerrewards.apiPure-Java contracts: the data-store interface and the kill/reward transport records.
common…slayerrewards.commonThe SQLite/MySQL/PostgreSQL/MongoDB backends and the central REST client.
bukkit…slayerrewards.bukkitShared Bukkit core: provider, mob-drop editor, the /slayerrewards command, and the kill listener.
spigotmc / papermc / foliamc…slayerrewards.<platform>Thin per-platform entry points.
engine…slayerrewards.engineThe universal jar for SpigotMC, PaperMC, and FoliaMC.

Command

/slayerrewards edit <mob>
Open the drop editor for a mob type (alias /slayer).
🔗

MCSlayerRewards runs alongside MCEconomy and MCIdentity. It declares both in the depend block of its plugin.yml, so they load first, and every payout is credited through the MCEconomy economy provider.
