Version 1.2.0

A permanent link to the MCSlayerRewards 1.2.0 release notes.

1.2.0 Latest 2026-07-21
Added
  • A master enable switch is now the first key in config.yml. It acts as a single on/off toggle for the whole plugin.
Changed
  • MCSlayerRewards now ships disabled by default (enable: false). Set it to true in config.yml to activate the plugin on your server.
  • When enable is false, the plugin shuts itself down during startup before touching any files.
Notes
  • No storage file is written while the plugin is disabled, and the SQLite slayerrewards.db is only created when database.type is sqlite — MySQL, PostgreSQL, MongoDB, and central backends never leave a .db behind.
  • Upgrading from 1.1.1: existing servers must set enable: true after the update to keep MCSlayerRewards running. No reward payouts or drop behaviour changed.
