MCEconomy

A per-identity, multi-currency economy for SpigotMC, PaperMC, and FoliaMC. Balances belong to the active MCIdentity identity, so every character is its own account.

io.github.mcclauneck.mceconomy SemVer 1.0.0 depends: MCIdentity
Change log Repository ↗

Highlights

🪙
Multi-currency. Define any number of currencies (coin, gold, point…), each transferable or account-locked.
🧍
Per-identity accounts. Balances are keyed to the active MCIdentity identity, never the raw player UUID.
💳
/balance & /pay. A GUI balance view and player-to-player transfers for transferable currencies.
🗄️
Five storage backends. SQLite, MySQL, PostgreSQL, MongoDB, or the central REST server.
🧩
Universal jar. One engine jar detects SpigotMC, PaperMC, or FoliaMC at enable time.

Module architecture

A multi-module Gradle build. The universal engine jar shades the shared modules and the thin per-platform entry points into one distributable.

ModuleJava packageResponsibility
apiio.github.mcclauneck.mceconomy.apiPure-Java contracts and models (accounts, currencies, wallets, the database interface).
common…mceconomy.commonThe MCEconomyProvider and the SQLite/MySQL/PostgreSQL/MongoDB/central backends.
bukkit…mceconomy.bukkitShared Bukkit core: the /balance and /pay commands, GUI, scheduler abstraction, events, and the MCIdentity listener.
spigotmc / papermc / foliamc…mceconomy.<platform>Thin per-platform entry points, bundled into the engine jar.
engine…mceconomy.engineThe universal central loader — the single distributed jar.

Storage backends

Choose a backend with database.type in config.yml. SQLite is the default and needs no configuration.

SQLite default

A zero-configuration local file (economy.db) inside the plugin data folder. Ideal for single-server setups.

MySQL / PostgreSQL

Pooled JDBC connections for a shared network database. Configure host, port, database, username, and password under the matching key.

MongoDB

A document store addressed by a connection URI and database name.

Central REST server

Point every game server on the network at one MCEconomy central deployment over HTTP so balances are shared network-wide.

Commands

/balance
Open the balance GUI for your active identity.
/pay <player> <currency> <amount>
Pay another player's active identity in a transferable currency.
🧭

Building a plugin on top of MCEconomy? Bind all stored data to the identity UUID (from MCIdentity's equip/unequip events), never the base player UUID — see the MCMarket and MCSlayerRewards pages for working examples.
